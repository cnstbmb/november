## Интересные сигналы из логов

### 1. Подозрительно для VPN/фильтрации/нестандартного DNS

- `startup.mobile.yandex.net -> 0.0.0.0`
- `startup-mobile.ap.yandex-net.ru -> 0.0.0.0`
- `startup.mobile.webvisor.com -> 0.0.0.0`
- `u.startup.mobile.webvisor.com -> 0.0.0.0`

Это обычно не "нормальный интернет", а признак DNS-блокировки, локального фильтра, AdGuard/Pi-hole, кастомного DNS или какого-то сетевого вмешательства. Такое устройство вполне может трактовать как сигнал "среда нестандартная".

### 2. Проверки доступности сети

- Есть `ICMP echo` на `8.8.8.8`
- Есть `ICMP echo` на `213.180.193.230`
- Есть `ICMP echo` на шлюз `192.168.1.1`

Это очень похоже на простую сетевую диагностику: "виден ли роутер", "виден ли Google", "виден ли Yandex".

### 3. Проверка DNS не только через роутер

- ТВ работает через локальный DNS `192.168.1.1:53`
- Но еще отдельно трогает `77.88.8.8:53/tcp`
- И пробует `192.168.1.1:853`, после чего получает `RST`

То есть устройство явно интересуется, какой DNS доступен, поддерживается ли DNS-over-TLS и что происходит с прямым доступом к внешнему DNS.

## Как по этому дампу устройство может палить VPN

### Наиболее вероятные способы

1. Серверная проверка IP/ASN/географии

Устройство ходит на много Yandex/Kinopoisk/Music API. На сервере очень просто увидеть:

- ASN провайдера
- датацентр/хостинг ли это
- страну и регион IP
- соответствует ли IP ожидаемому региону аккаунта/контента

Это самый вероятный механизм.

2. Проверка сетевой "нормальности"

По дампу видно признаки, которые могут считаться подозрительными:

- запросы, уходящие в `0.0.0.0`
- прямые проверки `8.8.8.8`, `77.88.8.8`, `213.180.193.230`
- попытка `DoT` на `192.168.1.1:853`

Если у тебя VPN/AdGuard/кастомный DNS, профиль поведения может отличаться от обычной домашней сети.

3. Сопоставление DNS и внешнего IP

Если устройство видит, что:

- DNS локальный/кастомный/блокирующий
- внешний IP относится к VPN/датацентру
- RTT/маршрут до Yandex и Google "странные"

то это уже хороший набор эвристик для метки "возможен VPN или сетевой прокси".
