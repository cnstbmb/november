# Как, вероятнее всего, детектится VPN в твоем кейсе

## Что видно по двум дампам

- С VPN (`dump.md`) устройство делает заметно больше control-plane и diagnostic трафика:
  - много коротких HTTPS-сессий к Yandex/Kinopoisk API;
  - DNS-запросы, ответы `0.0.0.0`;
  - `ICMP` до `8.8.8.8` и `213.180.193.230`;
  - попытки `77.88.8.8:53/tcp` и `192.168.1.1:853/tcp`.
- Без VPN (`dump2.md`) почти весь трафик это один большой поток с `37.9.93.206:443`, похожий на обычную доставку контента.
- Но `dump2.md` начинается уже на активной сессии, а не с холодного старта приложения, поэтому это не идеальный A/B.

## Самый вероятный механизм детекта

### 1. Серверный IP/ASN reputation

Самый сильный сигнал здесь не "VPN-протокол", а то, что внешний IP принадлежит датацентру.

- `timeweb.cloud` и `vdsina.ru` выглядят для сервиса как cloud/VPS egress, а не как обычный домашний ISP.
- Даже если RU-трафик выходит через Москву, сервис видит не только гео, но и ASN/hosting type/reputation.
- Для стриминга это типовой анти-abuse сигнал.

### 2. Несогласованность сетевого профиля одного устройства

У тебя схема `RU -> direct/RU exit`, `everything else -> DE exit`.

Для телевизора это может выглядеть неестественно:

- часть зависимостей приложения идет через RU egress;
- часть сторонних сервисов (например, Google/Firebase) может уйти через другой egress/ASN/страну;
- выбор CDN/edge для control-plane и content-plane становится другим, чем в обычной домашней сети.

Это не обязательно "палит VPN" само по себе, но может уводить сессию в другой серверный сценарий.

### 3. DNS как вторичный сигнал

В `dump.md` есть ответы `0.0.0.0` для:

- `startup.mobile.yandex.net`
- `startup-mobile.ap.yandex-net.ru`
- `startup.mobile.webvisor.com`
- `u.startup.mobile.webvisor.com`

Это похоже на `AdGuard Home`/локальную фильтрацию. Для приложения это признак "сеть нестандартная".

Отдельно видно:

- прямую проверку `77.88.8.8:53/tcp`;
- проверку `192.168.1.1:853/tcp` с `RST`;
- `ICMP` reachability probes.

Похоже, устройство или SDK проверяет, как именно у тебя устроен DNS/доступность сети.

## Что изменилось между VPN on/off

- С VPN нет явного большого контентного потока в фрагменте, зато много API и диагностических запросов.
- Без VPN появился выраженный bulk stream (`37.9.93.206:443`) и меньше служебного шума.
- Это больше похоже на разницу серверного routing/entitlement/CDN path, чем на локальный "VPN detector" на самом телевизоре.

## Что я бы тестировал в первую очередь

### 1. Для ТВ убрать split-path и сделать один консистентный egress

На время теста:

- весь трафик телевизора выпускать через один и тот же RU egress;
- не делить его на `RU -> one path`, `non-RU -> another path`.

Для стриминговых приложений консистентность пути часто важнее, чем "умная" маршрутизация.

### 2. Убрать AdGuard из цепочки именно для ТВ/Yandex/Kinopoisk

Минимум проверить без блокировок для:

- `*.yandex.net`
- `*.yandex.ru`
- `*.kinopoisk.ru`
- `*.webvisor.com`
- `startup.mobile.yandex.net`
- `startup-mobile.ap.yandex-net.ru`
- `startup.mobile.webvisor.com`
- `u.startup.mobile.webvisor.com`

Идея простая: для ТВ сделать максимально обычный DNS-профиль без `0.0.0.0`, rewrite и фильтрации.

### 3. Проверить сам класс exit IP

Если задача именно выглядеть как обычный RU-пользователь, московский VPS все равно остается датацентровым IP.

Практический вывод:

- RU geo само по себе недостаточно;
- hosting ASN очень вероятно является основным триггером.

### 4. Повторить A/B правильно

Нужен один и тот же сценарий:

1. холодный старт приложения;
2. открытие одного и того же тайтла;
3. `play`;
4. 20-30 секунд захвата.

И снять два дампа:

- с VPN;
- без VPN.

Параллельно полезно сохранить:

- внешний IP;
- ASN exit IP;
- какие DNS-ответы получил ТВ на стартовые домены.

## Короткий вывод

Наиболее вероятно, что тебя "детектят" не по WireGuard как таковому, а по совокупности:

- датацентровый exit IP;
- нестандартный DNS-профиль;
- неконсистентный путь для разных зависимостей одного и того же приложения.

Если привести для ТВ сеть к виду "один стабильный RU egress + обычный DNS без блокировок", это даст самый чистый тест и уберет самые сильные сигналы.
